# Проект: KnowledgeBridge

## Стиль ответов и взаимодействия
- Англицизмы оставляй только там, где это действительно стандартный термин разработки.

## Стиль кода (C# / .NET)
- Все XML и обычные комментарии пиши на русском языке.
- Предпочитай `internal` код вместо `public`, если нет явной необходимости открывать API шире.
- Все `public` методы и свойства должны быть покрыты XML-комментариями.
- Если XML-комментарий короткий, оформляй его в одну строку.

## Команды сборки, тестирования и разработки
- Дополнительные файлы для сборки и запуска помещай в `tmp/`.
